# Data Export (Export Hub) — Developer Implementation Guide

> **Feature type:** frontend
> **Last updated:** 2026-03-03

## Overview

The Export Hub is a right-side slide-in drawer that lets users export their expense data in multiple formats (CSV, JSON, PDF), send exports to cloud destinations (Google Sheets, Dropbox, OneDrive) or via email, configure automated scheduled exports, browse a timestamped export history, and generate shareable links with QR codes. All processing happens entirely in the browser — no server round-trips are made. Export history and schedule configuration are persisted to `localStorage`.

## How it fits into the architecture

The feature lives under `components/export-hub/` and is consumed by the Dashboard page (`app/page.tsx`). The Dashboard obtains the full expense list from the `useExpenses()` hook and passes it down as a prop. The drawer shell (`ExportHubDrawer`) owns the tab-routing state and orchestrates history refresh via a `refreshKey` counter. Persistence is handled by a dedicated store module (`lib/exportHubStore.ts`) that reads and writes two `localStorage` keys. The feature does not interact with `lib/store.ts` (the main expense CRUD store) directly — it only reads expenses passed in via props.

```
app/page.tsx
  └─ ExportHubDrawer (open, onClose, expenses)
       ├─ TemplatesTab   (expenses, onExportComplete)
       ├─ DestinationsTab (expenses, onExportComplete)
       ├─ ScheduleTab    ()
       ├─ HistoryTab     (refreshKey)
       └─ ShareTab       ()

lib/exportHubStore.ts  ←  HistoryTab, TemplatesTab, DestinationsTab, ScheduleTab
```

## Key types and interfaces

### `ExportRecord` — `lib/exportHubStore.ts`
```ts
interface ExportRecord {
  id: string;          // UUID generated by generateId()
  timestamp: string;   // ISO 8601 datetime of the export action
  templateName: string;// Human-readable template label, e.g. "Tax Report"
  destination: string; // 'Download' | 'Email' | 'Google Sheets' | 'Dropbox' | 'OneDrive'
  format: string;      // 'CSV' | 'JSON' | 'PDF'
  recordCount: number; // Number of expense rows included in the export
  status: 'success' | 'failed'; // Always 'success' in current implementation
}
```

### `ExportSchedule` — `lib/exportHubStore.ts`
```ts
interface ExportSchedule {
  enabled: boolean;                          // Whether automated exports are active
  frequency: 'daily' | 'weekly' | 'monthly';// Recurrence cadence
  destination: string;                       // Where to send scheduled exports
  templateId: string;                        // Which template to use (e.g. 'monthly-summary')
  nextRun: string;    // ISO 8601 datetime — next scheduled execution
  lastRun?: string;   // ISO 8601 datetime — last execution (undefined if never run)
}
```

### `Template` — `components/export-hub/TemplatesTab.tsx` (local)
```ts
interface Template {

  name: string;
  desc: string;
  format: 'CSV' | 'JSON' | 'PDF';
  color: 'emerald' | 'blue' | 'violet' | 'amber';
  Icon: React.ElementType;
  filter: (expenses: Expense[]) => Expense[]; // Data-selection logic for this template
}
```

### `Integration` — `components/export-hub/DestinationsTab.tsx` (local)
```ts
interface Integration {
  id: string;
  name: string;
  desc: string;
  Icon: React.ElementType;

```

## Function / component reference

### `ExportHubDrawer` — `components/export-hub/ExportHubDrawer.tsx`
```ts
function ExportHubDrawer({ open, onClose, expenses }: {
  open: boolean;
  onClose: () => void;
  expenses: Expense[];
}): JSX.Element
```
Root shell. Renders a full-screen overlay with a 64 px dark left-nav strip and a 540 px-max white content panel. Uses `translate-x-full` / `translate-x-0` CSS transitions for the slide animation. Owns `activeTab` and `historyRefreshKey` state. `handleExportComplete` is memoized with `useCallback`; it increments `historyRefreshKey` to trigger a `HistoryTab` re-read.

---

### `TemplatesTab` — `components/export-hub/TemplatesTab.tsx`
```ts
function TemplatesTab({ expenses, onExportComplete }: {
  expenses: Expense[];
  onExportComplete: (record: Omit<ExportRecord, 'id'>) => void;
}): JSX.Element
```
Two-phase UI. Phase 1: 2×2 grid of template cards. Phase 2 (after template selection): destination picker + export button. `filteredExpenses` is `useMemo`-derived. `runExport()` simulates 600 ms latency, performs the actual file generation, calls `addExportRecord()`, then calls `onExportComplete()`. Auto-resets to Phase 1 after 1 800 ms success display.

**Template filter functions:**
- `tax-report`: sorts all expenses by category then date
- `monthly-summary`: filters to `date >= getMonthStart(0)` (current month)
- `category-analysis`: returns all expenses unfiltered
- `quarterly-review`: filters to last 90 days

---

### `downloadFile` (module-private) — `TemplatesTab.tsx`, `DestinationsTab.tsx`
```ts
function downloadFile(content: string, filename: string, type: string): void
```
Creates a `Blob`, assigns it to a temporary `<a>` element via `URL.createObjectURL`, programmatically clicks it, then revokes the URL. Defined separately in both `TemplatesTab.tsx` and `DestinationsTab.tsx` (duplicated).

---

### `exportTemplatePDF` (module-private) — `TemplatesTab.tsx`
```ts
function exportTemplatePDF(templateName: string, expenses: Expense[]): void
```
Builds a self-contained HTML string with inline CSS styled in violet brand colors, opens it in a new tab via `window.open`, then calls `win.print()` after a 500 ms `setTimeout`.

---

### `openPrintPDF` (module-private) — `DestinationsTab.tsx`
```ts
function openPrintPDF(expenses: Expense[], title: string): void
```
Functionally identical to `exportTemplatePDF` but with a slightly simplified template and a 400 ms print delay. Used by the Destinations quick-download PDF path.

---

### `DestinationsTab` — `components/export-hub/DestinationsTab.tsx`
```ts
function DestinationsTab({ expenses, onExportComplete }: {
  expenses: Expense[];
  onExportComplete: (record: Omit<ExportRecord, 'id'>) => void;
}): JSX.Element
```
Three sections: (1) Cloud integrations with connect/disconnect toggle (simulated 1 400 ms connection delay). (2) Email send (simulated 900 ms, resets after 2 500 ms). (3) Quick download with CSV/JSON/PDF format picker and 500 ms simulated delay.

---

### `ScheduleTab` — `components/export-hub/ScheduleTab.tsx`
```ts
function ScheduleTab(): JSX.Element
```
Reads initial schedule via `getSchedule()` on mount. `update(patch)` merges partial updates and recalculates `nextRun` when `frequency` changes. `handleSave()` calls `saveSchedule(schedule)` and shows a success flash for 2 500 ms. The config form is visually disabled (opacity + `pointer-events-none`) when `schedule.enabled === false`.

---

### `HistoryTab` — `components/export-hub/HistoryTab.tsx`
```ts
function HistoryTab({ refreshKey }: { refreshKey: number }): JSX.Element
```
Calls `getExportHistory()` inside a `useEffect` keyed on `refreshKey`. Renders each `ExportRecord` with a destination icon, format badge, and relative timestamp (`timeAgo()`). `handleClear()` calls `clearExportHistory()` and resets local state.

---

### `ShareTab` — `components/export-hub/ShareTab.tsx`
```ts
function ShareTab(): JSX.Element
```
`generateLink()` creates a token via `generateId()` and sets a decorative URL (`https://expenses.app/share/{token}`). `copyLink()` uses `navigator.clipboard.writeText`. `FakeQRCode` renders a deterministic 21×21 SVG grid with real QR finder patterns and pseudo-random data modules seeded by the token's hash. No real data is transmitted.

---

### Store functions — `lib/exportHubStore.ts`

| Function | Signature | Description |
|----------|-----------|-------------|
| `getExportHistory` | `() => ExportRecord[]` | Parses `localStorage['export_hub_history']`; returns `[]` on parse error |
| `addExportRecord` | `(record: Omit<ExportRecord, 'id'>) => void` | Prepends new record, caps array at 50, writes back |
| `clearExportHistory` | `() => void` | Removes `export_hub_history` key |
| `getSchedule` | `() => ExportSchedule` | Parses `localStorage['export_hub_schedule']`; returns `makeDefault()` on miss/error |
| `saveSchedule` | `(s: ExportSchedule) => void` | Serializes schedule to `localStorage` |
| `nextRunISO` | `(freq: 'daily'\|'weekly'\|'monthly') => string` | Returns ISO datetime: +1d / +7d / first of next month |

## State management & data flow

```
1. app/page.tsx
   - useExpenses() → expenses[]
   - useState(false) → hubOpen
   - Renders <ExportHubDrawer open={hubOpen} expenses={expenses} />

2. ExportHubDrawer
   - useState('templates') → activeTab
   - useState(0) → historyRefreshKey
   - useCallback → handleExportComplete (increments historyRefreshKey)
   - Passes expenses + handleExportComplete to TemplatesTab / DestinationsTab

3. Export action (TemplatesTab or DestinationsTab)
   - Filters/sorts expenses locally
   - Generates file content in-memory
   - Triggers browser download or print dialog
   - Calls addExportRecord() → writes to localStorage['export_hub_history']
   - Calls onExportComplete() → bubbles to ExportHubDrawer

4. ExportHubDrawer.handleExportComplete
   - Increments historyRefreshKey

5. HistoryTab (useEffect on refreshKey)
   - Calls getExportHistory() → reads localStorage['export_hub_history']
   - Re-renders history list

6. ScheduleTab (independent)
   - Reads/writes localStorage['export_hub_schedule'] directly
```

## Edge cases & error handling

| Scenario | Handling |
|----------|----------|
| `window.open` blocked by popup blocker | Early return if `win` is null; no error shown to user |
| Empty filtered expense set | Export button is disabled when `filteredExpenses.length === 0` |
| `localStorage` parse failure | `try/catch` in `getExportHistory()` and `getSchedule()` returns safe defaults |
| `localStorage` quota exceeded | Not handled; `setItem` may throw silently |
| Cloud/email destinations | Fully simulated — no real network call is made |
| Schedule execution | UI only — no background job or service worker runs the schedule |
| Share URL | Decorative only (`https://expenses.app/share/{token}` does not exist) |
| `navigator.clipboard` unavailable (HTTP or old browser) | Unhandled — `copyLink()` will throw |
| `downloadFile` duplicate definition | `downloadFile` is copy-pasted into both `TemplatesTab.tsx` and `DestinationsTab.tsx` |

## Security considerations

- **Client-side only**: all expense data stays in `localStorage`; nothing is transmitted to any server.
- **CSV injection**: description fields containing formulas (e.g. `=CMD(...)`) are not sanitised before inclusion in CSV output. If the exported CSV is opened in Excel by a third party this could trigger formula injection. Mitigation: prefix suspicious cells with a single quote.
- **PDF/print HTML**: `exportTemplatePDF` and `openPrintPDF` interpolate `e.description` directly into an HTML string written to a new window via `document.write`. If a description contains `<script>` tags, those will execute in the context of the new blank window (isolated origin). Low severity in a single-user local app, but worth sanitising if multi-user is ever added.
- **Share link**: tokens are generated client-side with `generateId()` (UUID-like); the share URL is decorative and does not leak real data.
- **No authentication**: the app is single-user with no auth layer; all localStorage data is accessible to any script running on the same origin.

## Performance notes

- `filteredExpenses` in `TemplatesTab` is wrapped in `useMemo([selected, expenses])` — avoids re-filtering on unrelated renders.
- `handleExportComplete` in `ExportHubDrawer` is wrapped in `useCallback([])` — stable reference prevents unnecessary child re-renders.
- `FakeQRCode` grid computation is wrapped in `useMemo([token])` — the 21×21 hash calculation only reruns when the token changes.
- The drawer is **not unmounted** when closed — it stays in the DOM with `opacity-0 pointer-events-none`. This preserves tab state (e.g. a half-filled email field) across open/close cycles, at the cost of keeping all tab component trees alive.
- Export history is capped at 50 records in `addExportRecord` (`.slice(0, 50)`) to bound localStorage usage.
